关于ECMA Script中的对象

通常所说的 面向对象 的标志，是语言当中存在类的概念。虽然ES6当中引入了class关键字，但在本质上，它依旧是原型对象的 语法糖。

ECMA对于对象的定义是

无需属性的集合，其属性可以包含基本值、对象或函数

每一个对象都是基于一个引用类型创建的，这个引用类型可以是原生类型(如Array Buffer String)，也可以是自定义类型。

对象的创建方式

对象的创建方式有很多，JavaScript的函数本身也是一种对象，其基本数据类型 number string boolean也是对象，不同之处在于，基本类型是特殊的Wrapper Object -> 包装对象

原型

这种现象就是所谓的原型链，对于不存在于当前对象上的属性，将沿着对象的原型对象查找，如果查找存在，则输出。如果对象上定义了原型对象上已存在的属性，则将覆盖，jack的name age属性就是这个道理。因此，我们可以将原型对象看作是一份模板，将构造函数看作是一个机器。每次实例化这个机器就会copy一份该模板的复制品出来。特殊之处在于，复制品未重新定义的属性将沿用模板属性，新定义的属性将覆盖模板属性。因为我们没有重新定义greet方法，因此所有实例都将共用原型对象的greet方法。

检测某一个对象是不是另一个对象的原型可以使用isPrototypeOf方法
