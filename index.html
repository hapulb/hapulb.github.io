闲着翻译了EasyRTC的官方教程

January 13, 2018

概览

EasyRTC基于webRTC。WebRTC是W3C/IETF用于浏览器间实时音视频沟通以及数据传输的一个实现方案。WebRTC只需要一个轻量负荷的服务器就可以支持点对点(P2P)间的任何数据传输。

EasyRTC由客户端（浏览器端）的JS库与基于node.js的后端服务器组成。WebRTC已经被各个浏览器(google chrome, firefox, opera, etc)所支持，因此无需额外的浏览器插件。

Google Chrome对WebRTC的API有着最广泛的支持，Opera现在采用与Chrome相同的内核引擎，因此所有API行为与chrome基本一致。Firefox对WebRTC的Data Channel有着十分良好的实现，但仅提供了基础的视频功能。

一旦WebRTC被标准化，它将有着巨大的潜能为音视频会议，多用户游戏，以及许多其他的基于音视频，数据传输的应用提供支持。

如同其他的软件，强大的功能往往伴随着复杂的内部实现。WebRTC有着十分曲折的学习曲线，对开发人员不够友好。为了简化具体的开发流程，我们（Priologic）构建了EasyRTC框架。

构建一个基于WebRTC的应用通常有如下步骤。

  • 将本地摄像头、麦克风获取到的数据输出成media stream对象
  • 与信令服务器建立连接
  • 通过浏览器与目标用户建立p2p通信
  • 将media stream绑定到

通过使用EasyRTC，一些步骤可以被简化到一个简单的通信(call)当中，
能极大地简化开发流程，尤其是当开发人员需要投入更多的精力于多平台支持当中。

此文档是编写基于WebRTC应用的一个基本教程，但并未包含EasyRTC的所有API。

关于ECMA Script中的对象

通常所说的 面向对象 的标志，是语言当中存在类的概念。虽然ES6当中引入了class关键字，但在本质上，它依旧是原型对象的 语法糖。

ECMA对于对象的定义是

无需属性的集合，其属性可以包含基本值、对象或函数

每一个对象都是基于一个引用类型创建的，这个引用类型可以是原生类型(如Array Buffer String)，也可以是自定义类型。

对象的创建方式

对象的创建方式有很多，JavaScript的函数本身也是一种对象，其基本数据类型 number string boolean也是对象，不同之处在于，基本类型是特殊的Wrapper Object -> 包装对象

  • 对象字面量
  • 构造函数

面向对象编程的一个特色便是构造函数与析构函数。在JavaScript当中，没有类的概念，因此构造函数需要手动实现。

  • 早期

即是是所谓的工厂模式

原型

使用上述两种方式存在着诸多问题，例如，在传统的面相对象编程当中，所有实例拥有各自的属性，但共用方法。而在上述两种方式当中，greet()方法在每一次实例化对象时都会被创建，也就是说，每一次实例化一个Man对象，将同时实例化一个Function对象。对性能开销有着不小的影响。

因此引入原型模式

上述方式定义了Man的prototype属性，之后对Man实例化，如果输出jack，会得到

{ name: ‘jack’, age: 21 }

会发现jack不存在greet 属性，但是如果调用该方法

会得到

‘hey’

这种现象就是所谓的原型链，对于不存在于当前对象上的属性，将沿着对象的原型对象查找，如果查找存在，则输出。如果对象上定义了原型对象上已存在的属性，则将覆盖，jack的name age属性就是这个道理。因此，我们可以将原型对象看作是一份模板，将构造函数看作是一个机器。每次实例化这个机器就会copy一份该模板的复制品出来。特殊之处在于，复制品未重新定义的属性将沿用模板属性，新定义的属性将覆盖模板属性。因为我们没有重新定义greet方法，因此所有实例都将共用原型对象的greet方法。

检测某一个对象是不是另一个对象的原型可以使用isPrototypeOf方法

several concept

Promise 是异步编程的高级解决方案

简单来说，promise就是一个容器，里面保存着未来才会执行的事件

specialty

  • 不受外界影响

有三种状态 pending fulfilled rejected，只有异步操作的结果可以决定这个状态，其他任何操作都无法改变

  • 不变性

一旦状态确定，就不会再发生变化，任何时候都可以得到这个结果。

示例

resolve()将promise的状态从pending变为fulfilled， 并触发then()方法的第一个参数函数：reject()将promise从pending变为rejected，触发then()的第二个参数函数

promise 新建之后会立即执行

Promise.prototype.then()

  • 该方法返回一个新的Promise对象，因此可以使用链式写法

Promise.prototype.catch()

该方法是then(null, rejection)的别名，在promise状态变为rejected之后会调用
